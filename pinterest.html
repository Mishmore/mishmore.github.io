Pinterest

Brief

"Pinterest es una plataforma que permite a los usuarios crear boards temáticos donde pueden crear, guardar y compartir imágenes (pines) de distintas fuentes. ¿Su misión? Conectar a todos en el mundo a través de cosas que encuentren interesantes.

El proyecto Pinterest fue parte del Sprint 6 de Herramientas de Productividad en el Bootcamp de Laboratoria. El desafío fue recrear un board de un usuario real siguiendo el diseño original de la página. El desarrollo del proyecto fue en un equipo de 3 personas y su duración fue de 2 días.

Objetivos

- Replicar el diseño para desktop y mobile.

- Usar el API de Pinterest para traer información del board del usuario: arabelyuska/web-ui/

- Automatizar tareas con Gulp.

- Publicar el proyecto en Heroku.

Rol en el proyecto

Para garantizar la productividad del equipo en el tiempo disponible, me encargué de realizar las tareas para la actualización automatizada de los archivos de Sass, HTML5 y Javacript en carpeta publics. También adicioné una tarea para recargar el navegador una vez que haya detectado algún cambio en unos de los archivos.

Realicé la sección de la barra de navegación para desktop y mobile con el sistema de grillas del Framework Materialize. Tomé en cuenta la visibilidad de sus íconos en ambos casos y el efecto de traslado que tenía en el título del board al deslizar la rueda del mouse hacia abajo.

Modal y API.

Proceso

Herramientas utilizadas:

- Javascript - HTML5 - CSS - jQuery - Materialize - Gulp

Aprendizaje

Este reto me dio la posibilidad de ver desde una perspectiva más amplia la lógica detrás de una aplicación, pues logré recrear un flujo entre pantallas dependiendo de las decisiones que tome el usuario.

Además, realicé validaciones rigurosas, evaluando cada posible ingreso de data y permitiendo el envío de sólo la correcta. Este tipo de restricciones es fundamental para evitar errores en las peticiones y mantener la base de datos con información útil.

Logré también usar componentes para un mejor desempeño a nivel de carga, ya con ellos no es necesario repintar las páginas por cada interacción del usuario. Esto el bastante ímportante en aplicaciones móviles, donde el factor de tiempo es a veces determinante para la aceptación y uso de una app.
