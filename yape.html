Yape

Brief

"Yape es la nueva aplicación del Banco de Crédito del Perú, con la que se puede realizar transferencias de dinero desde el móvil. -

Con Yape podrás convertir tus planes en experiencias únicas, sin números de cuenta ni token." El sistema de registro de la aplicación Yape desarrolló como parte del proyecto final del sprint 5 del Bootcamp de Laboratoria. El desarrollo fue personal y tuvo una duración de 2 días y medio.

Objetivos

- Utilizar componentes para la creación de elementos de la aplicación y un cambio entre pantallas sin recargar la página.

- Validar todos los campos de registro para garantizar un envío de data libre de errores.

- Realizar de manera efectiva peticiones a la base de datos de la aplicación, utilizando el verbo de POST para enviar la data de registro de usuario.

Proceso

Herramientas utilizadas:

- Javascript - HTML5 - CSS - jQuery - Materialize

Aprendizaje

Este reto me dio la posibilidad de ver desde una perspectiva más amplia la lógica detrás de una aplicación, pues logré recrear un flujo entre pantallas dependiendo de las decisiones que tome el usuario.

Además, realicé validaciones rigurosas, evaluando cada posible ingreso de data y permitiendo el envío de sólo la correcta. Este tipo de restricciones es fundamental para evitar errores en las peticiones y mantener la base de datos con información útil.

Logré también usar componentes para un mejor desempeño a nivel de carga, ya con ellos no es necesario repintar las páginas por cada interacción del usuario. Esto el bastante ímportante en aplicaciones móviles, donde el factor de tiempo es a veces determinante para la aceptación y uso de una app.
